# VerseID вЂ” Frontend

TanStack Start (React SSR) В· Vite В· Tailwind CSS 4 В· TypeScript

This is the frontend for VerseID вЂ” a Bible verse identification app. It talks to the Django backend via a typed Axios client with automatic JWT refresh, and uses React Query for all server state.

---

## Requirements

- Node.js 18+
- [bun](https://bun.sh) (preferred вЂ” the project shipped with `bun.lock`) **or** npm

---

## 1 В· Install dependencies

```bash
cd verseid-frontend

# with bun (recommended)
bun install

# or with npm
npm install
```

---

## 2 В· Configure environment variables

```bash
cp .env.example .env
```

Open `.env` and fill in:

```dotenv
# URL of the Django backend API
VITE_API_BASE_URL=http://localhost:8000/api/v1

# Google OAuth Client ID вЂ” must match GOOGLE_CLIENT_ID in backend/.env
VITE_GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com

# VAPID public key for Web Push вЂ” generated by `python manage.py generate_vapid_keys`
# Must match VAPID_PUBLIC_KEY in backend/.env
VITE_VAPID_PUBLIC_KEY=your-vapid-public-key-here
```

> All three values are required for a fully working app. The app will start without them but Google sign-in and push notifications will not work.

---

## 3 В· Start the dev server

Make sure the backend is running first (`python manage.py runserver`), then:

```bash
# with bun
bun dev

# or with npm
npm run dev
```

App opens at `http://localhost:5173`.

---

## 4 В· Build for production

```bash
# with bun
bun run build

# or with npm
npm run build
```

Output goes to `.output/`. The app is a TanStack Start SSR application вЂ” it runs as a Node.js server, not a static file bundle. Start the production server with:

```bash
node .output/server/index.mjs
```

## 5 В· Google OAuth вЂ” browser origins

In [Google Cloud Console](https://console.cloud.google.com) в†’ APIs & Services в†’ Credentials в†’ your OAuth client, make sure these are listed under **Authorised JavaScript origins**:

```
http://localhost:5173
http://localhost:3000
https://your-production-domain.com
```

Without this Google Sign-In will throw an `origin_mismatch` error.

---

## 6 В· Push notifications

The service worker (`public/sw.js`) handles background push delivery. It is registered automatically on first load via `__root.tsx`. For push to work you need:

1. `VITE_VAPID_PUBLIC_KEY` set in `.env` (must match the backend's `VAPID_PUBLIC_KEY`).
2. A browser that supports the Push API (all modern browsers except Safari < 16).
3. The user to have granted notification permission in **Settings в†’ Enable push notifications**.

## 8 В· Key patterns

### Authentication
- `AuthContext` manages session state globally. `isReady` starts `false` while a stored access token is being verified вЂ” the landing page and static routes render immediately without waiting because the no-token path resolves synchronously.
- `/app/*` routes are guarded by `app.tsx` which redirects to `/auth` when `isReady && !user`.

### API calls
- All API calls go through `src/services/client.ts` (Axios instance). It automatically adds the `Authorization: Bearer <token>` header and refreshes the access token silently if a 401 is received (single-flight вЂ” concurrent 401s only trigger one refresh).

### Quota
- Free users get 20 searches/day. The identify endpoint returns `dailySearchesRemaining` on every response. A 429 response means the quota is exceeded вЂ” the client maps this to a structured `IdentifyQuotaExceeded` result that shows an upgrade prompt instead of a generic error.

### Voice search
- `useVoiceRecognition` wraps the browser's `SpeechRecognition` / `webkitSpeechRecognition` API. Interim transcripts display live as the user speaks. The final transcript is sent to `/search/identify/` вЂ” no audio ever leaves the device.

### Text-to-speech
- The "Listen" button on the Results screen uses `window.speechSynthesis` (browser built-in, free, works offline). Tapping again stops playback.
